Exakte .env-Einträge und Start-Kommandos (bore.pub)

Folge diesen Schritten genau. Danach kannst du in slAIdshow sofort Bilder über den bore‑Tunnel generieren.
1) Auf dem ComfyUI-Rechner

    ComfyUI lokal starten (nur local listen, bore greift auf 127.0.0.1 zu):

BASH

# im ComfyUI-Verzeichnis
python main.py --listen 127.0.0.1 --port 8188 --lowvram

    bore installieren und Tunnel starten (fixer Port + Secret empfohlen):

BASH

# macOS: brew install bore-cli
# Linux (cargo): cargo install bore-cli
# Windows: Binary von Releases

# Tunnel starten:
bore local 8188 --to bore.pub --port 28088 --secret "CHANGE_ME_VERY_STRONG"

Wichtig:

    Der Prozess muss laufen bleiben. Für Dauerbetrieb später als Service/Autostart einrichten.
    Wenn Port 28088 schon belegt ist, wähle einen anderen freien Port (z. B. 28188) und übernimm ihn in die .env unten.

Optionaler Schnelltest von einem anderen Rechner:
BASH

curl -I http://bore.pub:28088/

Eine HTTP-Antwort (auch 404/405) ohne Verbindungsfehler genügt als Zeichen, dass der Tunnel lebt.
2) Auf dem slAIdshow-Rechner: .env anpassen

Öffne die .env im slAIdshow-Projektordner und füge/ändere diese Zeilen:
DOTENV

# Image backend selection
APP_IMAGE_BACKEND=comfyui

# Allow remote ComfyUI backends (mandatory for bore)
APP_ALLOW_REMOTE_BACKENDS=1

# ComfyUI over bore (public endpoint)
APP_COMFY_HOST=bore.pub
APP_COMFY_PORT=28088

# Force remote-friendly image download mode
APP_COMFY_FORCE_VIEW_MODE=query

# Optional: increase backend timeout slightly for tunnel latency (if available in your version)
# APP_COMFY_MAX_WAIT_SEC=180

# Keep outputs local
APP_OUTPUT_DIR=./outputs/images

Speichern.

Starte slAIdshow neu:
BASH

source .venv/bin/activate
python app.py

Öffne die UI:

    http://127.0.0.1:8080

Generiere ein Bild.
3) Prüfen, ob alles verbunden ist

    In den slAIdshow-Logs solltest du NICHT mehr “comfy_unavailable” sehen.
    Falls doch, teste die Erreichbarkeit manuell vom slAIdshow-Rechner:

BASH

curl -I http://bore.pub:28088/
curl -s -X POST http://bore.pub:28088/prompt -H "Content-Type: application/json" -d '{"prompt":{}}'

    Eine 400/405 Antwort ist okay; entscheidend ist, dass keine Zeitüberschreitung/Connection refused kommt.

4) Häufige Stolpersteine

    APP_ALLOW_REMOTE_BACKENDS vergessen → Host-Policy blockt Remote-Zugriff.
    bore-Prozess beendet → Port ist tot; neu starten.
    Anderen Port genommen → in .env APP_COMFY_PORT entsprechend anpassen.
    Bilddownload hängt → setze APP_COMFY_FORCE_VIEW_MODE=query (bereits oben enthalten).
    Tunnel-Performance: bore ist unverschlüsselt und schlank; bei hohen Latenzen ggf. APP_COMFY_MAX_WAIT_SEC erhöhen.
